You are the world’s best UI/UX designer for marketplace and fintech platforms.

Design the Escrow Payment Flow and Payment Screen for my agricultural marketplace website.

IMPORTANT:
Do not create a new visual style.
Follow the existing design language, typography, color palette, spacing, and component styles from the current website UI so that the escrow system integrates seamlessly with the rest of the platform.

All elements must visually match the current website styling.

Screen Purpose

This screen is used when a buyer purchases crops from a farmer.

The payment must go through a secure escrow system where:

Buyer places order

Buyer pays into escrow

Seller ships goods

Buyer confirms delivery

Escrow releases funds to seller

The UI must clearly communicate this process to build trust and transparency.

Page Structure

Create a payment page layout with the following sections.

Page Header

Page title:

Secure Escrow Payment

Subtitle explaining the process:

“Your payment will be securely held in escrow until the goods are delivered and confirmed.”

Escrow Process Indicator

Add a progress tracker showing the escrow workflow.

Steps:

Order Placed

Payment Secured in Escrow

Seller Ships Goods

Buyer Confirms Delivery

Funds Released to Seller

The current step should be visually highlighted.

This component should clearly show where the transaction currently stands.

Order Details Section

Display the details of the order being paid for.

Include:

Product Name
Quantity
Price per unit
Total Price
Seller Name
Seller Location
Expected Delivery Time

The total payment amount should be clearly visible.

Escrow Explanation Section

Add a small section explaining how escrow protects both parties.

Show three short explanations:

Payment is held securely by the platform
Seller is notified once escrow payment is confirmed
Funds are released only after buyer confirms delivery

This section should increase user confidence in the transaction.

Payment Method Selection

Provide selectable payment options such as:

UPI
Credit or Debit Card
Net Banking

Only one option can be selected at a time.

The UI should clearly show which payment method is active.

Payment Summary

Display a summary of the payment breakdown.

Show:

Subtotal
Platform Fee
Total Amount Payable

The total amount should be visually emphasized.

Primary Payment Action

Add a primary action button:

“Pay Securely into Escrow”

When clicked, the system should initiate the escrow payment process.

Below the button include reassurance text explaining that funds are held securely until delivery confirmation.

Payment Confirmation Modal

Design a confirmation dialog that appears before payment is processed.

Display:

Order details
Total payment amount
Seller information

Confirmation message:

“Your payment will be held securely in escrow until delivery confirmation.”

Include two actions:

Confirm Payment
Cancel

Post-Payment Success State

Design a success screen after payment is completed.

Title:

Payment Successfully Secured in Escrow

Message explaining that the seller will now prepare the shipment.

Display next steps:

Seller preparing shipment
Delivery tracking will be available soon
Buyer will confirm delivery after receiving goods

Include a button:

View Order Status

Interaction Requirements

The UI should guide the user clearly through the escrow process.

Important states to include:

Before payment

Payment confirmation

Escrow secured

Awaiting shipment

Awaiting delivery confirmation

Funds released

These states will help the user understand the lifecycle of the transaction.

This escrow flow must integrate naturally with the existing marketplace UI and feel like a built-in part of the platform rather than a separate system.
